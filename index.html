Networking Fundamentals

What is Computer Networking?

Computer networking is the practice of connecting computers and other devices to share resources and information. When two or more devices communicate with each other, they form a network.

Real-World Example:

Think of a network like a postal system. Devices have addresses (IP addresses), and data is sent in packages (packets) that are routed through various post offices (routers) to reach their destination.

Key Networking Concepts

ISP (Internet Service Provider)

Your ISP is your gateway to the internet. Examples include Comcast, Verizon, Airtel, Jio, etc. They provide you with a public IP address that identifies your network on the internet.

How Data Travels

  1. You request a website (e.g., google.com) from your browser
  2. Your device creates a packet with your request
  3. The packet goes to your router
  4. The router sends it to your ISP using your public IP
  5. The ISP routes it through the internet to the destination server
  6. The response follows the reverse path back to your device

Wireless Technologies

Wi-Fi Standards

Wi-Fi has evolved through several standards, each offering improvements in speed, range, and security.

Standard Release Year Frequency Max Speed Key Features
802.11b 1999 2.4 GHz 11 Mbps First widely adopted standard
802.11a 1999 5 GHz 54 Mbps Less interference, faster than 802.11b
802.11g 2003 2.4 GHz 54 Mbps Backward compatible with 802.11b
802.11n (Wi-Fi 4) 2009 2.4/5 GHz 600 Mbps MIMO technology, wider channels
802.11ac (Wi-Fi 5) 2014 5 GHz 3.5 Gbps Wider channels, more spatial streams
802.11ax (Wi-Fi 6) 2019 2.4/5/6 GHz 9.6 Gbps OFDMA, better performance in dense areas

Wireless Security Protocols

Warning: Older security protocols like WEP are extremely vulnerable and should never be used.

WEP (Wired Equivalent Privacy)

INSECURE - Easily cracked within minutes using tools like Aircrack-ng. Uses RC4 encryption with weak initialization vectors.

WPA (Wi-Fi Protected Access)

WEAK - Improved over WEP but still vulnerable to attacks like KRACK (Key Reinstallation Attacks). Uses TKIP encryption.

WPA2 (Wi-Fi Protected Access 2)

SECURE - Currently the most widely used security protocol. Uses AES encryption and is considered secure when using a strong password.

WPA3 (Wi-Fi Protected Access 3)

MOST SECURE - The latest security protocol that provides individualized data encryption, protection against brute-force attacks, and easier connection for IoT devices.

Wireless Network Components

  • Access Point (AP): Device that allows wireless devices to connect to a wired network
  • Wireless Router: Combines AP, router, and often switch functionality
  • Wireless Network Interface Card (WNIC): Hardware that enables devices to connect to wireless networks
  • Antennas: Used to transmit and receive wireless signals

IP Addressing and Subnetting

What is an IP Address?

An IP address is a unique identifier assigned to each device on a network. It's like a home address for your computer on the internet or local network.

# Example IPv4 address
192.168.1.105

# Breakdown of the address
Network Part: 192.168.1
Host Part: 105

# Subnet Mask
255.255.255.0

# This means:
- Network: 192.168.1.0
- First usable IP: 192.168.1.1
- Last usable IP: 192.168.1.254
- Broadcast: 192.168.1.255

IPv4 vs IPv6

Feature IPv4 IPv6
Address Size 32 bits 128 bits
Address Format 192.168.1.1 2001:0db8:85a3:0000:0000:8a2e:0370:7334
Available Addresses ~4.3 billion 3.4×10³⁸ (essentially unlimited)
Security Optional (IPSec) Built-in (IPSec)

Subnetting

Subnetting is the process of dividing a network into smaller, more manageable sub-networks. This improves security and performance.

Subnetting Example:

If you have the network 192.168.1.0/24 (255.255.255.0), you can divide it into two subnets:

  • Subnet 1: 192.168.1.0/25 (255.255.255.128) - IPs 192.168.1.1 to 192.168.1.126
  • Subnet 2: 192.168.1.128/25 (255.255.255.128) - IPs 192.168.1.129 to 192.168.1.254

TCP Three-Way Handshake

The TCP three-way handshake establishes a reliable connection between two devices:

  1. SYN: Client sends a synchronization packet to the server
  2. SYN-ACK: Server responds with synchronization-acknowledgment
  3. ACK: Client sends acknowledgment to establish the connection

Network Security

Common Network Attacks

Man-in-the-Middle (MitM) Attacks

An attacker intercepts communication between two parties without their knowledge.

Denial of Service (DoS/DDoS)

Attackers overwhelm a network or service with traffic, making it unavailable to legitimate users.

Packet Sniffing

Capturing and analyzing network traffic to extract sensitive information like passwords.

Security Best Practices

  • Use strong, unique passwords for all network devices
  • Enable WPA2 or WPA3 encryption on wireless networks
  • Keep firmware and software updated
  • Disable unnecessary services and ports
  • Use a firewall to filter incoming and outgoing traffic
  • Implement network segmentation to limit breach impact

Network Scanning and Discovery

Nmap

Purpose: Network discovery and security auditing

Usage Examples:

# Basic network scan
$ nmap -sn 192.168.1.0/24

# Port scan with service detection
$ nmap -sV 192.168.1.1

# OS detection
$ nmap -O 192.168.1.1

$ nmap -A 192.168.1.1

Purpose: Network protocol analyzer for troubleshooting and analysis

Usage Examples:

$ wireshark -i wlan0

# Filter HTTP traffic only
http

ip.addr == 192.168.1.100

Right-click packet → Follow → TCP Stream

Network Troubleshooting

Common Network Issues

Connectivity Problems

  • No internet access: Check physical connections, router status, ISP outage
  • Intermittent connectivity: Check for interference, update drivers, check for overheating
  • Slow speeds: Check for bandwidth hogs, test with different devices

Wireless-Specific Issues

  • Weak signal: Reposition router, check for obstacles, consider range extenders
  • Interference: Change Wi-Fi channel, move away from other electronics
  • Authentication problems: Verify password, check security settings

Troubleshooting Methodology

  1. Identify the problem scope (single device vs entire network)
  2. Check physical connections and hardware status
  3. Verify IP configuration and connectivity
  4. Test DNS resolution
  5. Check for firewall or security software blocking connections
  6. Update drivers and firmware
# Basic network troubleshooting commands
$ ping google.com # Test connectivity
$ ip addr show # Check IP configuration
$ nslookup google.com # Test DNS resolution
$ traceroute google.com # Trace network path
$ netstat -tuln # Check listening ports
$ iwconfig # Check wireless settings

Advanced Networking Concepts

VLANs (Virtual LANs)

VLANs allow you to segment a physical network into multiple logical networks, improving security and performance.

VLAN Configuration Example:

A company might create separate VLANs for:

  • VLAN 10: Management (IT staff only)
  • VLAN 20: Employees
  • VLAN 30: Guests
  • VLAN 40: IoT Devices

Routing Protocols

Routing protocols determine how routers share information about network paths:

  • OSPF (Open Shortest Path First): Link-state protocol used in large enterprise networks
  • BGP (Border Gateway Protocol): Path-vector protocol used between ISPs
  • EIGRP (Enhanced Interior Gateway Routing Protocol): Cisco proprietary hybrid protocol

Software-Defined Networking (SDN)

SDN separates the network control plane from the data plane, allowing centralized management and programmability.

SDN Benefits:

  • Centralized network management
  • Dynamic resource allocation
  • Automated provisioning
  • Improved security through micro-segmentation

Practical Labs

Lab 1: Setting Up a Wireless Home Network

Objective: Configure a secure wireless home network with proper security settings.

Steps:

  1. Access your router's administration panel (usually 192.168.1.1 or 192.168.0.1)
  2. Change the default admin password
  3. Configure wireless settings:
    • Set SSID (network name) to something unique but not personally identifiable
    • Enable WPA2 or WPA3 security
    • Use a strong passphrase (at least 12 characters with mix of letters, numbers, symbols)
    • Select the appropriate wireless channel (use 1, 6, or 11 for 2.4GHz to minimize interference)
  4. Enable firewall and disable WPS (Wi-Fi Protected Setup) if possible
  5. Test connectivity from multiple devices

Lab 2: Analyzing Network Traffic with Wireshark

Objective: Capture and analyze network traffic to understand communication patterns.

Steps:

  1. Install Wireshark on your system
  2. Start a capture on your network interface
  3. Generate some network traffic (browse a website, use an app)
  4. Apply filters to focus on specific traffic:
    • http - Show only HTTP traffic
    • dns - Show only DNS queries
    • ip.addr == 8.8.8.8 - Traffic to/from Google DNS
  5. Examine packet details to understand the communication flow
  6. Look for any unencrypted data being transmitted

Lab 3: Network Mapping with Nmap

Objective: Discover devices and services on your network.

Steps:

  1. Determine your network range (ip addr show or ipconfig)
  2. Scan for active hosts: nmap -sn 192.168.1.0/24
  3. Perform service detection on discovered hosts: nmap -sV 192.168.1.1
  4. Identify operating systems: nmap -O 192.168.1.1
  5. Create a network map of all discovered devices and services

Ethical Note: Only perform these scans on networks you own or have explicit permission to test. Unauthorized scanning may be illegal.
